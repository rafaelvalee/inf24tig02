O que é o XML?

XML (eXtensible Markup Language) é uma linguagem de marcação usada para armazenar e transportar dados de forma estruturada, legível por humanos e máquinas. Muito usada para trocar informação entre sistemas, configurar aplicações e armazenar dados.

Estrutura básica de um ficheiro XML


<?xml version="1.0" encoding="UTF-8"?>
<linguagens>
  <linguagem>
    <nome>Python</nome>
    <dificuldade>Fácil</dificuldade>
  </linguagem>
</linguagens>
  • Só pode haver um elemento raiz (neste exemplo: <linguagens>).
  • Tags são sensíveis à ordem e ao fecho correto.
  • Podes criar qualquer estrutura de tags, mas deve ser consistente.

Funciona na mesma
Dá erro de validação

Atributos e elementos

Podes representar informação com elementos ou atributos.
Exemplo com atributos: 


<aluno nome="Rafael" idade="19" curso="Informática" />

Exemplo equivalente só com elementos: 


<aluno>
  <nome>Rafael</nome>
  <idade>19</idade>
  <curso>Informática</curso>
</aluno>

São mais fáceis de processar
Permitem guardar múltiplos valores com o mesmo nome
Tornam o documento mais compacto


Validação com XML Schema (XSD)

Um ficheiro XSD define as regras (estrutura, tipos, restrições) que um XML deve cumprir.
Exemplo de referência a um schema (namespace): 


<linguagens xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
  xsi:noNamespaceSchemaLocation="linguagens.xsd">
...
</linguagens>
  • Podes validar o teu XML com ferramentas como FreeFormatter XML Validator.

Garantir que o XML tem a estrutura certa
Guardar as imagens do site
Proteger o XML com senha


XML vs JSON

Atualmente, muitos sistemas usam JSON em vez de XML, mas XML continua fundamental em áreas como configuração, documentos oficiais, interoperabilidade e standards. 


// JSON equivalente ao exemplo anterior
{
  "aluno": {
    "nome": "Rafael",
    "idade": 19,
    "curso": "Informática"
  }
}

Mais legível por humanos
Aceita comentários no ficheiro
É menos compatível com browsers

Boas práticas & Dicas rápidas

  • Não uses espaços em nomes de tags ou atributos.
  • Não podes repetir nomes de atributos no mesmo elemento.
  • Evita usar acentos ou caracteres especiais nas tags.
  • Sempre fecha todas as tags, mesmo que estejam vazias (<exemplo />).
  • Valida sempre o teu XML com uma ferramenta apropriada.
